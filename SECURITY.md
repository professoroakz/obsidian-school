# Security Policy

## Supported Versions

Currently supporting the latest version on the main branch.

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please follow these steps:

1. **DO NOT** open a public issue
2. Email the maintainer with details about the vulnerability
3. Include steps to reproduce if possible
4. Allow time for a fix to be developed and deployed

## Security Best Practices

When contributing to this repository:

- Never commit sensitive information (passwords, API keys, tokens)
- Use environment variables for configuration
- Keep dependencies up to date
- Follow secure coding practices
- Review changes before committing

## Response Timeline

- Initial response: Within 48 hours
- Status update: Within 7 days
- Fix deployment: Depends on severity and complexity

## Disclosure Policy

- Security issues are fixed privately
- Once fixed, a security advisory will be published
- Credit will be given to reporters (if desired)

## Contact

For security concerns, please create a security advisory on GitHub or contact the repository owner.
